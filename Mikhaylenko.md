# Мануал по работе с Git

*Краткое руководство по работе с* **GIT** *с использованием возможностей языка программированния .*

*Постепенно файл будет дополняться.*

## Базовые команды Git
*Для возможности внесения commit'ов необходимо зарегистрировать себя в git. для этого используются команды:*

git config --global user.name "имя пользователя" - ввод имени пользователя git.

git config --global user.mail "адрес почты" - ввод почты пользователя git.

git status - просмотр статуса git

![Пример использования команды git status](GitStatus.JPG)

*Для того чтобы внести commit в уже открытый файл необходимо* **сохранить файл**, *после чего внести следующие команды:*

git add FileName - выбор файла, в который вносятся commit'ы. Имя файла можно ввести руками, либо выбрать название файла с помощью клавиши **TAB** .

git commit -m "NameOfCommit" - добавление commit'а в файл. Название commit'а нужно обязательно указывать в " ". 

## Особенности работы с репозиториями
*Для работы с репозиториями необходимо в папке (**директории**) проекта созадть папку .FolderName (например .git), где будет хранится конфигурация нашего репозитория, после чего необходимо ввести команду:*

git init - команда инициализации нашего проекта 

## Перемещение между коммитами
> *Для просмотра всех commit'ов используется команда:*

git log

![Пример использования команды git log](GitLog.JPG)

> *Для вывода списка всех веток в виде "дерева" используется команда git log с ключом:*

git log --graph

![Пример использования graph](GitBranches.JPG)

> *Для переключения между commit'ами используется команда:*

git checkout CommitName

# Работа с ветками
> *Для создания ветки используется команда:*

git branch NameOfBranch - где NameOfBranch имя ветки которое мы задаем самостоятельно

> *Для удаления ветки используется команда:*

git branch -d NameOfBranch - где NameOfBranch имя ветки

> *Для слияния веток используется команда:*

git merge NameOfBranch - где NameOfBranch имя ветки, из которой необходимо сделать перенос. **Важно!** При слиянии веток необходимо находится в ветке **master** 

## Для простого добавления файлов в код, можно в проекте создать файл **.gitignore**, в который вписываем названия файлов с их расширениями.

![Пример файла .gitignore](GitIgnore.JPG)

Clear

# Работа с удаленными репозиториями

> При работе с чужим репозиторием обязательно нужно добавить его в свой аккаунт на GitHub (сделать **fork**), для дальнейшей возможности работы с ним и внесения изменений и коммитов.

fork - клонирование репозитория в свой аккаунт GitHub.

> Есть возможность скопировать удаленный репозиторий к себе на локальный компьютер, для этого используется команда:

git clone *url* - где url ссылка на клонируемый репозиторий.

> Команды для работы с удаленными репозиториями:

git push - отправить изменения, внесенные на локальную версию, в удаленный репозиторий

git pull - получить/скачать в локальную версию актуальное состояние удаленного репозитория
